### ১। টাইপস্ক্রিপ্ট এ Interface ও Type এর মধ্যে পার্থক্য ###

# ব্যবহারিক উদ্দেশ্য
Interface অবজেক্ট বা ক্লাসের গঠন বর্ণনার জন্য ব্যবহার করা হয়।
Type সাধারণত  ইউনিয়ন, ইন্টারসেকশন, প্রিমিটিভ, টিউপল, কন্ডিশনাল ইত্যাদি ধরনও প্রকাশ করতে পারে।
# ডিক্লারেশন মার্জিং
Interface পুনরায় খুলে অতিরিক্ত প্রপার্টি যোগ করা যায়।
Type একবার ডিফাইন হলে বন্ধ হয়ে যায়, পরে নতুন করে প্রপার্টি যোগ করা যায় না।
# এক্সটেন্ড করার পদ্ধতি
Interface সরাসরি এক্সটেন্ড ব্যবহার করে।
Type ইন্টারসেকশন টাইপ এর মাধ্যমে প্রসারিত হয়।
# ফ্লেক্সিবিলিটি
Interface দীর্ঘমেয়াদী এপিআই, মডেল এবং অবজেক্ট ওরিএন্টেড প্রগ্রামিং ভিত্তিক কোডের জন্য আরও সুবিধাজনক।
Type জটিল, কম্বিনেশন-ভিত্তিক, টুলস টাইপ সিস্টেমে বেশি শক্তিশালী।
# কমিউনিটি ও কোডিং কনভেনশন
Interface সাধারণত পাবলিক এপিআই, লার্জ স্কেল অ্যাপ ও আর্কিটেকচার ডিজাইন এ বেশি প্রেফার করা হয়।
Type ফাংশনাল প্যাটার্ন, রিইউজেবল ইউটিলিটি টাইপিং এবং কমপ্লেক্স টাইপ লজিক এ বেশি জনপ্রিয়।


### ২। TypeScript এ any, unknown এবং never — পার্থক্য ###

# any
any এমন একটি টাইপ যা টাইপস্ক্রিপ্ট এ টাইপ চেকিংকে সম্পূর্ণ বন্ধ করে দেয়।
ভেরিয়েবলের মধ্যে কী আসছে, কী রিটার্ন করছে, কী অপারেশন হচ্ছে কিছুই নির্দিষ্ট করা হয় না।
এটি ব্যবহার করলে কম্পাইল টাইম এ এরর ধরতে পারে না, টাইপ সেফটি নষ্ট হয় এবং বড় কোডবেসে বাগ তৈরি হওয়ার সম্ভাবনা বৃদ্ধি পায়।
# unknown 
unknown, any-এর থেকে নিরাপদ।
এতে আপনি কিছু অ্যাসাইন করতে পারবেন, কিন্তু ব্যবহার করার আগে টাইপ চেক করতে বাধ্য হবেন। 
এর ফলে কম্পাইল টাইম এ এরর প্রতিরোধ হয়, টাইপ চেকিং নিশ্চিত হয়, কোড হয় আরও নিরাপদ ও মেইনটেইনেবল।
# never
never এ কোনো ফলাফল, রিটার্ন, বা ভ্যালু কখনও থাকবে না।
এটি ব্যবহার করা হয় এমন ফাংশনে, যেখানে সবসময় এরর থ্রো করা হয়, ইনফিনিট লুপ চলে,এক্সহস্টিভ টাইপ চেকিং করা হয়। 
এটি কোডকে লজিক্যালি সঠিক রাখতে সাহায্য করে এবং অপ্রত্যাশিত শাখা কাজ করছে কি না নিশ্চিত করে। never বেশি ব্যবহার হয় লজিক ভ্যালিডেশন এ।
